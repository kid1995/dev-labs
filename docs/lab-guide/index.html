Quick Start

Prerequisites

  • Docker Desktop with 8 GB+ memory (Settings → Resources → Memory)
  • Java 21 (Temurin): $JAVA_HOME=/Users/.../temurin-21.0.8/Contents/Home
  • Node.js 22.x + npm

First Time Setup

# 1. Build internal Java libraries (jwt-adapter, elpa4-model)
cd java_libs/jwt-adapter && ./gradlew publishToMavenLocal -Pversion=2.5.0 -x check
cd java_libs/elpa4-shared-lib/elpa4-model && ./gradlew publishToMavenLocal -Pversion=1.1.1 -x check

# 2. Build applications
cd dev-labs/src/hint-backend && ./gradlew build -x check -x dependencyCheckAnalyze
cd dev-labs/src/dlt-manager && ./gradlew build -x check -x dependencyCheckAnalyze -x rewriteRun
cd dev-labs/src/dlt-manager/frontend && npx ng build --configuration production

# 3. Build Docker images
docker build -f docker/hint-backend.Dockerfile -t lab/hint-backend:latest src/hint-backend/
docker build -f docker/dlt-backend.Dockerfile -t lab/dlt-backend:latest src/dlt-manager/
docker build -f docker/dlt-frontend.Dockerfile -t lab/dlt-frontend:latest src/dlt-manager/frontend/

# 4. Start everything
cd dev-labs && docker compose up -d

# 5. Copy mavenLocal to Jenkins (after Jenkins starts)
docker exec lab-jenkins mkdir -p /root/.m2/repository/de/signaliduna/elpa
docker cp ~/.m2/repository/de/signaliduna/elpa/jwt-adapter lab-jenkins:/root/.m2/repository/de/signaliduna/elpa/
docker cp ~/.m2/repository/de/signaliduna/elpa/elpa4-model lab-jenkins:/root/.m2/repository/de/signaliduna/elpa/
docker exec lab-jenkins bash -c 'apt-get update -qq && apt-get install -y -qq docker.io'

Subsequent Starts

# With aliases (from anywhere):
lab-up          # start all, wait for healthy, print URLs
lab-down        # stop all (data preserved)

# Without aliases:
cd dev-labs && docker compose up -d

Key Pattern: si-button requires native <button>

The design system's ::slotted(button) CSS selector ONLY targets a native <button> element slotted inside <si-button>. Using <span> results in an unstyled 19px-tall element.

<!-- WRONG: unstyled -->
<si-button si-variant="primary">
  <span>Click me</span>
</si-button>

<!-- CORRECT: styled by design system -->
<si-button si-variant="primary">
  <button>Click me</button>
</si-button>

Reference: Storybook → Button → Primary

Shell aliases are defined in ~/.zshrc. They let you manage the lab from anywhere without cd-ing into dev-labs/.

These aliases are already added to your ~/.zshrc. If starting fresh, add:

# Dev-Labs shortcuts (add to ~/.zshrc)
export DEV_LABS="$HOME/Downloads/github-buffer/dev-labs"

Local DNS (Pretty URLs)

Instead of remembering port numbers, use human-readable hostnames via a reverse proxy.

How it works

  1. /etc/hosts maps *.labs.local / *.elpa.local / *.si.local to 127.0.0.1
  2. An nginx reverse proxy container (lab-proxy) on port 80 routes by hostname to the right container
  3. All hostnames are defined in one file: config/proxy/routes.yaml

Mistakes & Lessons Learned

Every wrong decision and error encountered during lab setup, in chronological order.

Error #1 downloadSuppression fails at Gradle config time

What happened: hint build.gradle calls downloadSuppression() during Gradle configuration phase — tries to reach git.system.local. Even -x dependencyCheckAnalyze doesn't skip it.

Fix: Wrapped in try-catch. Lesson: Gradle config-phase code runs unconditionally.

Error #2 Version mismatch for mavenLocal dependencies

What happened: jwt-adapter published as 2.5.1-SNAPSHOT but dlt-manager needs exact 2.5.0. Same for elpa4-model.

Fix: Published with -Pversion=2.5.0. Lesson: Always check exact version required before publishing.

Error #3 Modified original source code

What happened: Changed combine-hint/hint/build.gradle directly instead of working on a copy.

Fix: Reverted, cloned to dev-labs/src/, applied changes only there. Lesson: Never modify original project repos.

Error #4 Kafka healthcheck path wrong

What happened: kafka-broker-api-versions.sh not in PATH inside Apache Kafka container. Exit code 127.

Fix: Full path /opt/kafka/bin/kafka-broker-api-versions.sh. Lesson: Always check tool paths inside containers.

Error #5 Kafka internal/external listener confusion

What happened: Single PLAINTEXT listener with localhost:9092 — backends inside Docker can't reach it.

Fix: Added EXTERNAL listener on port 19092 for host, PLAINTEXT on 9092 for inter-container. Lesson: Docker networking requires separate listeners.

Error #6 Bitbucket DC — HV000149 validation error

What happened: Bitbucket DC setup wizard throws HV000149: An exception occurred during message interpolation on admin user creation. Hibernate Validator bug with newer JDK.

Decision: Tried multiple fixes (JVM args, version downgrade, memory increase). All failed. Lesson: Don't fight enterprise software bugs. Switched to Gitea.

Error #7 GitLab CE — wrong platform + OOM + password rejection

What happened: (1) amd64 image on ARM Mac, (2) OOM with 3.8GB Docker memory, (3) password rejected as "commonly used".

Decision: Tried ARM image, increased memory to 8GB, random password — still took 10+ min to start. Lesson: GitLab CE is too heavy for a lab. Use Gitea.

Error #8 Frontend: Unexpected token 'export'

What happened: Angular emits scripts-*.js with defer but the file contains ES module export syntax.

Fix: Strip the export{...} via sed + load as classic script. Lesson: Check script type before assuming module/non-module.

Error #9 Frontend: CustomElementRegistry double registration

What happened: Both scripts bundle and Angular chunks register same web components → "name already used" error.

Fix: Guard customElements.define to be idempotent. Lesson: Web component registration is global and irreversible.

Error #10 OIDC: invalid issuer trailing slash

What happened: Frontend expects http://localhost:8180/realms/lab/ but Keycloak returns http://localhost:8180/realms/lab (no trailing slash).

Fix: Removed trailing slash from AUTH_URL in docker-compose. Lesson: OIDC issuer URLs must match exactly.

Error #11 OIDC: Invalid scopes: openid si_common

What happened: Keycloak client didn't have si_common as a default scope.

Fix: Added to defaultClientScopes in lab-realm.json. Lesson: OIDC scopes must be configured per-client, not just per-realm.

Error #12 si-button unstyled — invented own CSS

What happened: Button rendered as unstyled 19px text. Tried multiple hacks: idempotent define guard, script load order, shadow DOM style injection via MutationObserver. Used wrong colors (dark blue bg instead of teal).

Root cause: Template used <span> instead of <button>. Design system uses ::slotted(button) CSS selector.

Fix: Changed <span>Login with SSO</span> to <button>Login with SSO</button>. Removed all patches.

Lesson: Always check the Storybook for correct component usage before inventing workarounds. The si-patch.js hack produced wrong colors and wrong architecture.

Error #13 .gitignore too aggressive in dlt-manager clone

What happened: Original .gitignore excluded gradle/, *.gradle, config/, *.md, *.txt — Jenkins couldn't find Gradle wrapper, settings.gradle, or Java source files in config/ package.

Fix: Rewrote .gitignore to only exclude build artifacts. Lesson: Always verify git content after pushing, especially with aggressive gitignore.
