# Start Here

Read these docs before working on dev-labs. They contain hard-won knowledge from building and debugging the lab — things not obvious from the code alone.

## Reading order

1. **01-architecture.md** — What this lab simulates and how components map to corporate
2. **02-copsi-pipeline.md** — The CoPSI deployment pattern (the main thing being tested)
3. **03-gotchas.md** — Non-obvious problems that will waste hours if you don't know them
4. **04-workarounds.md** — Active workarounds and why they exist
5. **05-technical-debt.md** — Known issues to fix when time allows
6. **06-user-intent.md** — What the user is trying to achieve and design decisions

| Understand the project | 01-architecture.md |
